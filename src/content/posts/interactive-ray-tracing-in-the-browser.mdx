For years this blog has been static HTML generated from Ghost. The problem: when I
want to *show* an idea — a 3D render, a shader, a simulation that reacts to your
input — a screenshot doesn't cut it. So I'm moving to Astro, where an article is
still plain markdown, but I can drop a real, interactive component right into the
middle of the prose.

Here's the whole point, live. Drag the sliders — the scene re-renders every frame
in your browser, no server involved:

<RaytracerCanvas />

That's a fragment-shader raymarcher: a signed-distance sphere on a checkered plane,
Phong-shaded with a soft shadow, the whole frame computed per-pixel on your GPU.
The **light angle**, **sphere radius**, and **camera distance** sliders feed
straight into shader uniforms. You can even drag on the canvas to orbit the light.

The key idea is Astro's *islands* model: the text around this widget ships as
static HTML with zero JavaScript, so the page stays fast. Only the canvas
"hydrates" and runs code. A long article with one interactive figure pays the JS
cost of exactly that one figure — nothing more.

Mechanically, embedding it was two lines: an `import` at the top of this file and a
`<RaytracerCanvas />` tag here in the body. The component itself is a single
`.astro` file holding the markup, the controls, and the WebGL `<script>`. Future
posts can pull in Three.js scenes, react-three-fiber, live plots, or anything else
the same way — while every migrated post stays clean, dependency-free markdown.
